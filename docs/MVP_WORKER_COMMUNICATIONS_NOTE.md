# MVP Worker Communications Note

For MVP launch, worker communications are intentionally simplified.

## Active worker communication surfaces

- Notifications (`/c1/workers/notifications`) as the in-app event feed and deep-link surface
- SMS for urgent operational updates
- Email for formal/richer communication
- Direct recruiter contact from assignment/job detail surfaces

## Removed from worker MVP

- Worker Inbox as a primary product surface

Legacy worker Inbox URLs are redirected away from Inbox for MVP.

## Future naming reservation

If a threaded communication experience returns in future phases, product naming should be:

- `Support`

and not:

- `Inbox`
